# AI 操作可见化使用说明

当你让 AI 通过 agent-browser 操作某个 Profile 时，ProfilePilot 可以在浏览器页面上显示一个状态条：你能看到 AI 正在做什么、做到哪一步，并且可以随时停止 AI、自己接管这个浏览器。

典型场景：你把一个已登录的隔离 Profile 交给 AI 去填写网页、查资料或跑测试。过程中你打开同一个 Chrome 窗口，就能在页面右上角看到 AI 的当前动作；如果发现它走偏了，点击状态条里的「停止并接管」，Chrome 页面会留在原处，AI 驱动连接会被停止。

## 开启和关闭

这个功能默认开启。

在 ProfilePilot 主界面选择一个独立 Profile，右侧详情面板会出现「Agent 连接」区域。里面有一行「AI 操作可见化」，右侧开关显示「已开启」或「已关闭」：

- 开启后，agent-browser 等 AI 工具连接这个 Profile 的 CDP 端口并操作普通网页时，页面会显示状态条。
- 关闭后，ProfilePilot 会移除页面上的状态条，并停止继续注入和更新。

如果你看不到这个开关，先确认选中的是带 CDP 地址的独立 Profile。系统 Chrome Profile 不支持由 ProfilePilot 开端口式 CDP 调试。

## 状态条怎么看

状态条默认显示在网页右上角，可以拖动标题栏改变位置，也可以点「−」收起成一个小圆点。小圆点可再次展开。

- 标题：显示谁在操作。单会话时类似「AI 正在操作 · Claude Code」或「AI 正在操作 · Codex」；多会话共用同一 Profile 时显示「AI 正在操作 · 2 个会话」。
- 当前动作：显示 AI 最近的浏览器动作，例如打开站点、点击元素、填写输入框、截图、读取页面结构、滚动或刷新。没有解析到具体动作时，会显示「AI 正在操作浏览器」。
- 当前步骤：如果 AI 会话里有计划或待办，会显示当前正在执行的步骤。
- 进度：如果能识别总步骤数和已完成步骤数，会显示「第 x/y 步」和下方进度条。
- 下一步：如果会话计划里有后续步骤，会显示「下一步：……」。
- 已运行时长：显示 ProfilePilot 本轮看到这个 AI 会话后已经运行多久。
- 会话列表：当同一个 Profile 上有多个 AI 会话时，会列出每个会话的工具名、标题或项目，以及最近活动时间。状态条主体优先展示最近活跃的会话。
- AI 最近说：如果能读取 Claude Code 或 Codex 的会话记录，会折叠显示 AI 最近一条文字消息。
- 「在 ProfilePilot 中查看」按钮：状态条标题栏里的 ⧉ 按钮会把 ProfilePilot 主窗口切到对应 Profile。

状态条会根据页面背景和系统外观自动切换亮色/深色。文案跟随 ProfilePilot/系统语言：中文环境显示中文，其他环境显示英文。

## 怎么接管

### 在浏览器页面接管

1. 点击状态条底部的「停止并接管」。
2. 3 秒内再次点击「再点一次确认接管」。
3. ProfilePilot 会停止对应的 AI 驱动连接，Chrome 窗口和当前网页不会关闭。

多会话时，按钮会显示「全部停止并接管」。这会停止同一个 CDP 端口上的 agent-browser 驱动连接；如果没有 agent-browser，则停止 ProfilePilot 识别到的 AI 驱动客户端。

接管成功后，页面状态条会切到「已接管，AI 已停止操作」，并在几秒后自动收起。ProfilePilot 主界面会显示 toast 和「最近接管」记录；「接管历史」弹窗最多保留最近 50 条。系统通知可用时也会弹出通知；如果 mini 窗口正在显示，ProfilePilot 会避免重复通知。

### 在 ProfilePilot 主界面接管

在 Profile 详情面板的「Agent 连接」区域，如果当前有 AI 驱动连接，会出现「接管」按钮。点击后会弹出确认框，说明将停止几条 AI 驱动连接。确认后，ProfilePilot 会断开这些连接，Chrome 窗口仍会保留。

这个入口适合你已经在 ProfilePilot 里看到某个 Profile 正被 AI 驱动、想直接从主界面停止它。当前实现中，主界面行内「接管」会给出操作结果提示；「最近接管 / 接管历史」主要记录从网页状态条发起的接管。

## 前提与边界

- 只对带 CDP 端口启动的 Profile 生效。agent-browser 操作的通常就是这类 Profile。
- ProfilePilot 管理的独立 Profile 支持这一能力；由其他工具启动、但能被 ProfilePilot 发现的外部 CDP 实例也可能显示活动状态，但生命周期不完全由 ProfilePilot 管理。
- 系统 Chrome Profile 没有由 ProfilePilot 开出的 CDP 地址，不会显示这个状态条。
- 状态条只注入普通网页。`chrome://`、DevTools、扩展页、`edge://`、`about:chrome`、`view-source:chrome` 等内部页面不会显示。
- 升级后如果某些旧标签页没有立刻出现状态条，刷新或导航一次通常会恢复。
- 如果 AI 工具自动重启了驱动进程，接管后 ProfilePilot 可能会在下一轮状态刷新时再次识别到它，状态条也会重新变为运行中。

## 隐私与安全

状态条运行在浏览器的隔离执行环境中，并挂在 closed Shadow DOM 里。普通网页 JavaScript 不能读取状态条内容，也不能伪造「停止并接管」信号。

接管信号只接受 ProfilePilot 注入的隔离环境发出的调用，因此恶意网页不能冒充你的点击来触发接管。状态条也会标记为 `aria-hidden`，减少 AI 通过页面可访问性快照把它当成网页内容的概率。

需要注意：状态条是视觉元素，截图里会看到它；网页脚本也可能观察到页面里存在一个固定位置的外层节点。它的目标是让人能看见和接管 AI 操作，不是隐藏视觉痕迹。

## FAQ

**没看到状态条怎么办？**

先检查 ProfilePilot 右侧详情面板「Agent 连接」里的「AI 操作可见化」是否开启。再确认这个 Profile 是用「CDP启动」打开的独立 Profile，并且 agent-browser 或其他 AI 工具正在连接它的 CDP 地址。最后确认当前标签页是普通网页；内部页面不会显示。仍然没有时，刷新页面或导航一次。

**AI 在操作，但状态条不显示步骤怎么办？**

这通常是会话文件没有被识别到。ProfilePilot 会读取 Claude Code 的 `cc-...` 会话和 Codex 的 `cx-...` 会话来解析步骤、下一步和最近消息；如果会话文件缺失、命名不匹配或暂时不可读，状态条会降级为只显示工具名、项目、会话和「AI 正在操作浏览器」等基础信息。

**接管后想让 AI 继续怎么办？**

重新从你的 AI 工具里启动或连接 agent-browser，让它再次连接这个 Profile 的 CDP 地址即可。新的驱动连接被识别后，状态条会重新显示运行状态。

**接管会关闭 Chrome 或丢失当前网页吗？**

不会。接管停止的是持有 CDP 连接的 AI 驱动进程或客户端，不会关闭 Chrome 进程，也不会主动关闭当前标签页。

**能不能临时隐藏状态条？**

可以。点击标题栏的「−」会把状态条收成小圆点；点击小圆点可展开。这个设置保存在当前页面会话里。要完全关闭功能，请在 ProfilePilot 的「Agent 连接」区域关闭「AI 操作可见化」开关。
